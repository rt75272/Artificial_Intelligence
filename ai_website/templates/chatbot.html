Rule-Based vs ML NLP

This chatbot uses handcrafted rules — not learned weights. When you send a message, the bot matches your text against a library of regular expressions to find the closest intent, then returns a pre-written response.

This is 100% deterministic: the same input always gives the same output. Modern LLMs (like ChatGPT) use neural networks instead, enabling generalization.

Deterministic No Training Regex Patterns

How a Message is Processed

1 Normalize: Input is lowercased and stripped of punctuation.
2 Tokenize: The sentence is split into individual word tokens.
3 Match: Each pattern (regex) in the rule set is tested against the input.
4 Respond: The best matching rule's response is returned.
5 Fallback: If no rule matches, a default "I don't understand" reply is sent.

Key Takeaways

Rules are brittle — A small typo or unusual phrasing can break pattern matching, while ML models can handle variation gracefully.
Determinism is a feature — In safety-critical applications (banking, medical), predictable, auditable responses are preferred over probabilistic ones.
Tokenization first — Breaking text into tokens is the foundation of virtually all NLP pipelines, whether rule-based or neural.
Intent ≠ semantics — Matching "hi" to a greeting intent is intent recognition. Understanding the full meaning of a sentence requires much deeper models.
