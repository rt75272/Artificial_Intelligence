What are Cellular Automata?

A cellular automaton is a grid of cells, each in one of a finite set of states, that evolves over discrete time steps according to local rules. The Game of Life, invented by mathematician John Conway in 1970, uses a 2D grid of alive (1) or dead (0) cells.

The remarkable thing is that its four rules are so simple a child can learn them — yet the patterns they generate can be infinitely complex.

Zero-Player Game Discrete Time Local Rules

The Four Rules

1 Underpopulation: A live cell with fewer than 2 live neighbors dies.
2 Survival: A live cell with 2 or 3 live neighbors survives to the next generation.
3 Overpopulation: A live cell with more than 3 live neighbors dies.
4 Birth: A dead cell with exactly 3 live neighbors becomes alive.

Grid edges wrap around, so neighbors across an edge count.

Key Takeaways

Emergence — Complex, ordered behavior (gliders, oscillators, guns) arises from four local rules with no global coordination. This mirrors phenomena in physics, biology, and economics.
Sensitivity to initial conditions — A single changed cell can lead to a completely different eventual state, echoing chaotic systems in nature.
Turing completeness — The Game of Life can simulate any computation, including simulating itself. This is proven using glider-gun based logic gates.
Parallel computation — All cells update simultaneously each generation. This "massively parallel" structure is why cellular automata are studied in parallel computing research.
