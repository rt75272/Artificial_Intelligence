How Decision Trees Work

A decision tree makes predictions by asking a sequence of yes/no questions about the input features. Each question (a "split") divides the data into two groups, recursively, until each leaf is mostly one class.

The tree learns which feature and threshold to split on by choosing the split that most reduces impurity — how mixed the classes are in each resulting group.

Gini Impurity

The most common split criterion is the Gini impurity, which measures how often a randomly chosen element would be incorrectly labeled:

Gini = 1 − Σ pᵢ²

A pure node (all one class) has Gini = 0. A maximally mixed node (50/50) has Gini = 0.5. The tree picks the split that minimizes the weighted average Gini of the two children.

Key Takeaways

Splits are axis-aligned — Standard decision trees only split along one feature at a time, creating rectangular decision regions. This means they can struggle with diagonal boundaries.
Depth controls overfitting — A very deep tree will perfectly memorize training data (overfitting). Limiting max_depth forces the tree to generalize rather than memorize.
Decision trees are interpretable — Unlike neural networks, you can trace exactly why a prediction was made by following the yes/no path from root to leaf.
Random Forests extend this — By averaging many randomized decision trees, Random Forests dramatically reduce variance while preserving the decision tree's ability to capture nonlinear patterns.
