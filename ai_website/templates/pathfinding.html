What is A*?

A* (pronounced "A-star") is a graph search algorithm that finds the shortest path from a start node to a goal node. It combines the advantages of Dijkstra's algorithm (guaranteed shortest path) with a heuristic that guides the search toward the goal efficiently.

Unlike BFS which expands all neighbors equally, A* prioritizes nodes that look most promising according to the formula below.

f(n) = g(n) + h(n)
Informed Search Optimal Complete

The Algorithm Step-by-Step

1 Open list: Start with just the start node. Each node stores g(n) — actual cost from start — and h(n) — estimated cost to goal.
2 Pick best: Pop the node with the lowest f(n) = g(n) + h(n) from the open list.
3 Expand: Add its walkable neighbors to the open list with updated g-costs.
4 Closed list: Move the current node to the closed list so it's not re-expanded.
5 Repeat until the goal is popped from the open list (path found!) or the list is empty (no path).

The Heuristic h(n)

The heuristic estimates how far node n is from the goal. For this grid demo, we use the Manhattan distance — the sum of horizontal and vertical steps needed if there were no walls.

h(n) = |x_goal − x_n| + |y_goal − y_n|

A heuristic must be admissible (never overestimates the true cost) to guarantee an optimal path. Manhattan distance is admissible because diagonal shortcuts don't exist in a 4-directional grid.

Key Takeaways

Heuristic quality matters — A better heuristic means fewer nodes are expanded, making the search faster. A perfect heuristic would go directly to the goal.
Admissibility guarantees optimality — If the heuristic never overestimates, A* is guaranteed to find the shortest possible path.
BFS is a special case — When h(n) = 0, A* becomes identical to Dijkstra's algorithm; when g(n) = 0, it becomes a greedy best-first search.
Real-world use — A* powers GPS navigation, game AI movement, and robot motion planning where finding the shortest path is critical.
