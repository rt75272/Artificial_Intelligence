What is a Random Forest?

A Random Forest builds many decision trees, each trained on a random subset of data. Each tree votes on the outcome, and the majority vote wins. This ensemble approach reduces overfitting and outperforms any single tree.

prediction = majority_vote(tree₁, tree₂, … treeₙ)

Feature Extraction

The model never looks at the actual characters — it only sees numerical features derived from them:

lengthTotal character count
has_lowerContains a–z?
has_upperContains A–Z?
has_digitContains 0–9?
has_specialContains !@#…?
unique_charsDistinct characters

Password Entropy

Entropy measures unpredictability. A password with more character types and greater length has a vastly larger search space for attackers:

entropy = length × log₂(charset size)

A 12-char password using all character types has ~78 bits of entropy — effectively uncrackable by brute force.

Key Takeaways

Ensemble > single tree — Random Forest averages out individual tree errors, giving much more reliable predictions than any single decision tree.
Features, not raw data — The model never sees your password as text; it only sees 7 numeric measurements. This is called feature engineering.
Length beats complexity — A 20-character lowercase phrase is far harder to crack than an 8-character "complex" password.
Uniqueness matters — Repeating characters like "aaaaaaa" reduces the effective charset size and lowers entropy drastically.
